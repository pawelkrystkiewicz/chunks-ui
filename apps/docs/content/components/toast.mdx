# Toast

A notification system built on Base UI's `Toast` primitive. Toasts slide in from the bottom-right corner and auto-dismiss. The `useToast` hook triggers them from anywhere within the provider.

## Import

```tsx
import { Toast } from "chunks-ui";
```

## Setup

Wrap your app (or the relevant subtree) with `Toast.Provider` and place `Toast.Viewport` where the toasts should render:

```tsx
export default function RootLayout({ children }) {
  return (
    <Toast.Provider>
      {children}
      <Toast.Viewport />
    </Toast.Provider>
  );
}
```

## Basic Usage

Use `Toast.useToast()` to get the `add` function and trigger notifications:

```tsx
function SaveButton() {
  const { add } = Toast.useToast();

  return (
    <button onClick={() => add({ title: "Saved", description: "Your changes have been saved." })}>
      Save
    </button>
  );
}
```

## With Action

Pass `actionProps` to render a labelled action button inside the toast:

```tsx
const { add } = Toast.useToast();

add({
  title: "Message deleted",
  description: "The message has been removed from your inbox.",
  actionProps: {
    children: "Undo",

  },
```

## Icons and per-type styling

`chunks-ui` doesn't ship any icons or semantic type styles itself ‚Äî bring your own. Three per-toast slots let you plug into whatever icon library and design tokens your project uses:

| Field | Purpose |
|---|---|
| `icon` | Component rendered in the leading slot. Receives a `className` prop pre-populated with default sizing (`size-5 shrink-0 self-start`). |
| `iconClassName` | Extra classes merged onto `icon` ‚Äî use for color. |
| `className` | Extra classes merged onto `Toast.Root` ‚Äî use for accent bars, borders, background tints. |

For the simple case, pass a lucide component directly:

```tsx
import { CheckCircle2 } from 'lucide-react'

add({
  title: "File saved",
  icon: CheckCircle2,
  iconClassName: "text-success",
})
```

For per-type styling, define a data-only map and spread it into `add()`:

```tsx
import { AlertTriangle, CheckCircle2, Info, XCircle } from 'lucide-react'

const TYPE_STYLES = {
  primary:     { icon: Info,          iconClassName: "text-primary",     className: "border-l-4 border-primary" },
  success:     { icon: CheckCircle2,  iconClassName: "text-success",     className: "border-l-4 border-success" },
  destructive: { icon: XCircle,       iconClassName: "text-destructive", className: "border-l-4 border-destructive" },
  warning:     { icon: AlertTriangle, iconClassName: "text-warning",     className: "border-l-4 border-warning" },
} as const

function notify(type: keyof typeof TYPE_STYLES, title: string) {
  add({ title, ...TYPE_STYLES[type] })
}
```

No factory functions, no closures ‚Äî the icon references are stable, so React reconciles them cleanly. Skipping all three fields gives you a plain, unstyled toast. Everything the library ships in Toast is additive to Base UI.

## Dismissable Toasts

By default `Toast.Viewport` does **not** render a close button ‚Äî toasts auto-dismiss after the provider's `timeout`. To let users dismiss them manually, pass `dissmissable` on the viewport:

```tsx
<Toast.Provider>
  {children}
  <Toast.Viewport dissmissable />
</Toast.Provider>
```

Every toast rendered by that viewport now has a `√ó` button in the top-right corner that dismisses it via Base UI's `Toast.Close`.

### Reacting to Close

If you need to run a side effect when the user dismisses a toast ‚Äî tracking analytics, offering an undo stack, syncing with a backend ‚Äî pass `onClose`. It receives the close button's click event. Passing `onClose` also implicitly enables the close button, so you don't need to also set `dissmissable`:

```tsx
<Toast.Viewport
  onClose={(event) => {
    analytics.track('toast_dismissed', { id: event.currentTarget.dataset.toastId })
  }}
/>
```

`onClose` runs *in addition to* Base UI's built-in dismissal ‚Äî it's for side effects, not for preventing the toast from closing.

## Compound Parts

| Part | Description |
|---|---|
| `Toast.Provider` | Context provider. Must wrap any component that calls `useToast`. |
| `Toast.Viewport` | Fixed-position container that renders all active toasts. |
| `Toast.Root` | Individual toast wrapper with slide/fade enter/exit transitions. |
| `Toast.Title` | Bold toast heading. |
| `Toast.Description` | Supporting text in `text-muted-foreground`. |
| `Toast.Close` | Icon button that dismisses the toast. |
| `Toast.Action` | Optional action button rendered inside the toast. |
| `Toast.useToast` | Hook that returns `{ add, toasts }` for the nearest provider. |

## Toast.useToast ‚Äî `add` Options

| Option | Type | Description |
|---|---|---|
| `title` | `string` | Toast heading text |
| `description` | `string` | Supporting body text |
| `icon` | `ComponentType<{ className?: string }>` | Component rendered in the leading slot. Library passes default sizing via `className` ‚Äî pass a lucide/heroicons component directly |
| `iconClassName` | `string` | Extra classes merged into the `icon`'s `className` (use for color) |
| `className` | `string` | Extra classes merged into `Toast.Root` (use for per-toast borders, accents, backgrounds) |
| `actionProps` | `{ children: ReactNode; onClick: () => void }` | Props for the action button |
| `timeout` | `number` | Auto-dismiss delay in milliseconds (provider default applies if omitted) |
